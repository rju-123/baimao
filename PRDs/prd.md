白帽子产品需求文档 (PRD)

## 一、产品概述

### 1.1 产品名称
白帽子业务平台

### 1.2 产品定位
一个面向B2B业务的销售订单管理系统，为销售人员提供订单创建、管理、履约跟踪以及积分激励等完整功能的移动端应用平台。同时为合作伙伴提供开票信息提交和审核通道。

### 1.3 产品目标
- 为销售人员提供便捷的订单创建和管理工具
- 为合作伙伴提供开票信息提交和审核流程，审核通过后可进行销售工作
- 通过积分商城、优惠券和邀请码机制提升用户活跃度和忠诚度
- 提供知识库帮助销售人员学习产品知识和销售技巧

### 1.4 目标用户
- **销售人员**：负责创建销售订单、管理订单流程、跟踪订单状态、使用积分商城
- **合作伙伴**：需要提交企业开票信息并通过后台审核，审核通过后即可进行销售工作

## 四、详细功能需求

### 4.1 登录认证模块

#### 4.1.1 登录页面
**页面路径**：`/login`

**功能描述**：
- 提供快捷登录和手机验证码登录两种方式
- 必须勾选用户协议和隐私政策才能登录
- 登录成功后跳转到角色选择页面

**页面元素**：
- 应用Logo（蓝色背景，手机图标）
- 应用名称："任务管理平台"
- "快捷登录"按钮（蓝色主按钮）
- "通过手机验证码登录"按钮（白色边框按钮）
- 用户协议勾选框："登录即表示同意《用户协议》和《隐私政策》"

**交互逻辑**：
1. 用户点击"快捷登录"时，检查是否已勾选协议
2. 如未勾选，弹出 alert 提示："请先同意用户协议和隐私政策"
3. 如已勾选，跳转到角色选择页面 (`/role-selection`)
4. 点击"通过手机验证码登录"跳转到验证码登录页面 (`/verification-login`)

**业务规则**：
- 协议必须勾选才能登录
- 两种登录方式效果相同，只是验证方式不同

---

#### 4.1.2 手机验证码登录页面
**页面路径**：`/verification-login`

**功能描述**：
- 提供手机号输入和短信验证码验证功能
- 支持发送验证码和60秒倒计时

**页面元素**：
- 返回按钮（左上角，返回登录页）
- 页面标题："手机验证码登录"
- 手机号输入框（带手机图标）
- 验证码输入框
- "发送验证码"按钮（带倒计时功能）
- "登录"按钮（蓝色主按钮）
- 用户协议勾选框

**交互逻辑**：
1. 输入11位手机号，自动校验格式
2. 点击"发送验证码"：
   - 校验手机号格式
   - 发送验证码（当前为模拟）
   - 按钮变为"60秒后重新发送"并开始倒计时
   - 倒计时结束后恢复"发送验证码"文字
3. 输入6位验证码
4. 勾选用户协议
5. 点击"登录"：
   - 校验手机号、验证码、协议勾选状态
   - 验证通过跳转到角色选择页面

**业务规则**：
- 手机号必须是11位数字
- 验证码60秒内只能发送一次
- 必须勾选协议才能登录

#### 4.1.3 角色选择页面
**页面路径**：`/role-selection`

**功能描述**：
- 用户选择身份角色（合作伙伴或销售人员）
- 根据选择的角色进入不同流程

**页面元素**：
- 页面标题："请选择您的身份"
- 副标题："选择后可随时在设置中更改"
- 两个角色选择卡片

**角色选项**：

1. **合作伙伴**
   - 图标：握手图标（Handshake）
   - 标题："我是合作伙伴"
   - 描述："提交开票信息、管理合作"
   - 选择后行为：
     - 保存角色到 localStorage: `userRole = 'partner'`
     - 跳转到开票信息提交页面 (`/partner/invoice-submission`)

2. **销售人员**
   - 图标：公文包图标（Briefcase）
   - 标题："我是销售人员"
   - 描述："创建订单、管理客户"
   - 选择后行为：
     - 保存角色到 localStorage: `userRole = 'sales'`
     - 跳转到公司选择页面 (`/sales/company-selection`)

**交互逻辑**：
- 点击角色卡片即选择该角色
- 角色信息保存到本地存储
- 根据角色跳转到对应的下一步流程

**业务规则**：
- 合作伙伴必须先提交开票信息并审核通过后才能进行销售工作
- 销售人员必须先选择所属公司才能开始工作

### 4.2 合作伙伴模块

#### 4.2.1 开票信息提交页面
**页面路径**：`/partner/invoice-submission`

**功能描述**：
- 合作伙伴提交企业开票信息和资料
- 后台审核通过后，用户获得销售人员权限，可以进行销售工作
- 当前实现为模拟审核失败状态

**页面布局**：
分为两个状态显示：

**状态1：信息填写状态**

页面元素：
- 页面标题："提交开票信息"
- 副标题："请上传清晰的开票信息照片或文件"
- 企业信息表单
- 开票资料上传区
- "提交"按钮

**企业信息表单字段**：
1. 企业名称（必填，文本输入框）
2. 联系人姓名（必填，文本输入框）
3. 注册地址（必填，文本输入框）
4. 联系电话（必填，11位手机号验证）
5. 税号（必填，文本输入框）
6. 开户银行（必填，文本输入框）
7. 银行账号（必填，文本输入框）

**开票资料上传区**：
- 上传区域（虚线边框）
- 上传图标
- 提示文字："点击上传开票资料"
- 支持格式："支持 JPG、PNG 格式"
- 图片预览（上传后显示）
- 文件名显示

**状态2：审核失败状态**

页面元素：
- 警告图标（橙红色圆形背景）
- 标题："审核失败"
- 提示文字："请联系工作人员了解详情"
- "我知道了"按钮

**交互逻辑**：

**信息填写流程**：
1. 用户填写所有企业信息字段
2. 点击上传区域，触发文件选择器
3. 选择图片文件（JPG/PNG）
4. 显示图片预览和文件名
5. 点击"提交"按钮：
   - 校验所有必填项是否填写
   - 校验是否已上传图片
   - 如有未填写项，弹出提示
   - 全部填写完整后提交


**审核失败状态**：
1. 显示审核失败提示
2. 点击"我知道了"按钮
3. 跳转到登陆页面

**业务规则**：
- 所有字段均为必填项
- 联系电话必须是11位数字
- 必须上传开票资料图片才能提交
- **重要**：实际业务中，审核通过后用户应该获得销售人员权限，需要：
  - 跳转到公司选择页面 (`/sales/company-selection`)
  - 选择公司后即可开始销售工作
- 当前为演示目的，模拟审核失败状态

**未来改进**：
- 需要对接真实的后台审核流程
- 审核中状态展示
- 审核通过后的权限开通
- 审核失败原因详情展示
- 支持重新提交

**页面路径**：`/sales/created`

**功能描述**：
- 展示销售人员创建的所有订单
- 支持按状态筛选订单
- 作为销售人员的首页（底部导航栏第一个Tab）

**页面布局**：
- 顶部分段控件（状态筛选）
- 订单列表（卡片形式，垂直滚动）
- 底部导航栏（4个Tab）

**订单状态分类**：
分段控件选项：
1. **全部** - 显示所有状态的订单
2. **签约中** - 签约阶段的订单
3. **履约中** - 正在履约的订单
4. **已完成** - 已完成的订单
5. **已取消** - 已取消的订单

**订单卡片信息**：
- 订单编号（如：ORD-2026-101）
- 订单标题/服务名称（大字号）

  - 履约中：绿色
  - 已完成：灰色
  - 已取消：红色
- 订单金额（右侧，大字号，如：¥85,000）
- 创建时间（底部，灰色小字）

**模拟订单数据示例**：
```
签约中：
- ORD-2026-101 企业网络安全咨询服务 某制造企业 ¥85,000

派单中：
- ORD-2026-102 数据中心安全评估 某能源集团 ¥120,000

履约中：
- ORD-2026-103 云平台安全加固 某互联网公司 ¥95,000

已完成：

已取消：
```

3. 点击订单卡片跳转到订单详情页面 (`/sales/order-details/:id`)
4. 支持下拉刷新订单列表
5. 空状态显示友好提示（如"暂无订单"）

**底部导航栏**：
- Tab 1: 我的订单（当前页，选中状态）
- Tab 2: 下单
- Tab 3: 积分商城
- Tab 4: 个人中心

**业务规则**：
- 订单按创建时间倒序排列（最新的在前）
- 订单列表实时同步，创建新订单后自动刷新
- 不同状态的订单数量显示在分段控件上（可选）

## 六、业务流程图

### 6.1 用户注册和登录流程
```
开始
  ↓
登录页
  ├→ 快捷登录（勾选协议）→ 角色选择
  └→ 手机验证码登录 → 输入手机号 → 发送验证码 → 输入验证码 → 角色选择
  
角色选择
  ├→ 选择"合作伙伴"
  │   ↓
  │   开票信息提交页
  │   ↓
  │   填写企业信息 + 上传资料
  │   ↓
  │   提交
  │   ↓
  │   后台审核（当前模拟为审核失败）
  │   ↓
  │   审核通过 → 公司选择页 → 选择公司 → 进入销售工作区
  │   审核失败 → 提示失败 → 跳转下单页（演示）
  │
  └→ 选择"销售人员"
      ↓
      公司选择页
      ↓
      选择公司 或 添加公司
      ↓
      进入销售工作区（显示底部导航栏）
```

### 6.2 销售下单流程
```
下单页（浏览产品列表）
  ↓
点击产品卡片
  ↓
产品详情页（查看详细信息）
  ↓
点击"立即下单"
  ↓
购买确认页
  ├→ 调整购买数量
  ├→ 上传电子签名
  ├→ 选择优惠券（可选） → 可用优惠券弹窗 → 选择优惠券 → 返回并应用折扣
  ↓
点击"提交订单"
  ↓
校验必填项
  ↓
提交成功
  ↓
订单详情页（显示订单信息）
  ↓
订单出现在"我的订单"列表中
```

### 6.3 积分兑换流程
```
积分商城页（浏览商品列表）
  ↓
点击"立即兑换"
  ↓
检查积分是否足够
  ├→ 积分不足 → 提示"积分不足"
  └→ 积分足够 → 兑换确认页
      ↓
      选择兑换数量
      ↓
      【实体商品】选择收货地址 → 地址选择页 → 返回
      【虚拟商品】无需地址
      ↓
      确认积分消耗和剩余
      ↓
      点击"确认兑换"
      ↓
      扣除积分
      ↓
      【实体商品】
        ↓
        创建兑换记录（待发货状态）
        ↓
        提示"兑换成功，等待发货"
      
      【虚拟商品】
        ↓
        生成券码
        ↓
        弹出券码弹窗（显示券码）
        ↓
        用户复制券码
        ↓
        关闭弹窗
    
```

后台管理系统功能需求
### 3.1 合作企业管理

#### 3.1.1 功能描述
管理与公司合作的企业客户信息，包括企业资质审核、开票信息管理、关联销售人员查看等功能。页面中展示企业ID、企业名称、联系人、联系电话、注册地址、开票信息图片。查看的弹窗页面中可查看全部信息并且选择审核状态。

#### 3.1.2 核心字段

| 字段名称 | 字段类型 | 是否必填 | 说明 |
|---------|---------|---------|------|
| 企业ID | String | 自动生成 | 格式：ENT001 |
| 企业名称 | String | 是 | 企业全称 |
| 联系人 | String | 是 | 主要联系人姓名 |
| 联系电话 | String | 是 | 11位手机号 |
| 注册地址 | String | 是 | 企业注册地址 |
| 纳税人识别号 | String | 是 | 统一社会信用代码 |
| 开户银行 | String | 是 | 企业开户银行名称 |
| 银行账户 | String | 是 | 企业银行账号 |
| 开票信息图片 | File | 否 | 支持上传开票资质图片 |
| 审核状态 | Enum | 是 | 待审核/已通过/已驳回 |

#### 3.1.3 功能操作
- **新增企业**：填写企业基本信息，上传开票资质图片
- **编辑企业**：修改企业信息（审核通过后可编辑）
- **删除企业**：软删除，可恢复
- **查看详情**：查看企业完整信息及开票资质图片
- **查看下属销售**：查看该企业的销售人员列表（含角色：管理员/销售）
- **审核管理**：
  - 待审核企业需人工审核
  - 支持通过/驳回操作
  - 驳回需填写驳回原因
- **搜索筛选**：
  - 按企业名称搜索
  - 按审核状态筛选

#### 3.1.4 业务规则
- 新增企业默认为"待审核"状态
- 审核通过后才能关联订单

### 3.2 销售人员管理

#### 3.2.1 功能描述
管理企业的销售团队成员，区分管理员和普通销售角色，追踪销售人员的订单业绩。可通过操作对角色信息进行编辑以及设置角色身份。

#### 3.2.2 核心字段

| 字段名称 | 字段类型 | 是否必填 | 说明 |
|---------|---------|---------|------|
| 销售ID | String | 自动生成 | 格式：S001 |
| 姓名 | String | 是 | 销售人员姓名 |
| 手机号 | String | 是 | 11位手机号，唯一 |
| 所属企业 | String | 是 | 关联企业ID |
| 角色 | Enum | 是 | 管理员/销售 |
| 进行中订单数 | Number | 自动计算 | 统计该销售员的进行中订单 |
| 状态 | Enum | 是 | 在职/离职 |

#### 3.2.3 功能操作
- **新增销售**：录入销售人员基本信息，分配角色
- **编辑销售**：修改销售信息和角色
- **删除销售**：软删除（有历史订单的不可删除）
- **设置状态**：设置为离职状态
- **查看订单**：查看该销售的所有订单列表
- **搜索筛选**：
  - 按姓名/手机号搜索
  - 按所属企业筛选
  - 按角色筛选
  - 按状态筛选

#### 3.2.4 业务规则
- 每个企业有一个管理员角色
- 手机号全局唯一
- 离职销售不能接新订单

### 3.5 订单管理

#### 3.5.1 功能描述
网络安全服务订单的全生命周期管理，包括订单创建、合同上传、派单、履约追踪等。

#### 3.5.2 核心字段

| 字段名称 | 字段类型 | 是否必填 | 说明 |
|---------|---------|---------|------|
| 订单ID | String | 自动生成 | 格式：ORD001 |
| 产品 | String | 是 | 关联产品ID |
| 金额 | Number | 是 | 订单金额 |
| 接单员 | String | 是 | 关联销售人员ID |
| 白帽子 | String | 派单后必填 | 关联白帽子ID |
| 订单状态 | Enum | 是 | 见下方状态说明 |
| 电子合同 | File | 否 | PDF/Word格式，含上传时间 |

#### 3.5.3 订单状态流转

```
待签约 (pending_contract)
    ↓ [上传合同]
待履约 (pending_fulfillment)
    ↓ [分配白帽子开始服务]
履约中 (in_progress)
    ↓ [完成服务]
已完成 (completed)

任何阶段都可 → 已取消 (cancelled)
```

#### 3.5.4 状态说明

| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| 待签约 | 订单已创建，等待合同上传 | 上传合同、取消订单 |
| 待履约 | 合同已上传，分配白帽子，开始服务 | 分配白帽子、取消订单 |
| 履约中 | 服务进行中 | 完成服务 |
| 已完成 | 服务已完成 | 查看详情 |
| 已取消 | 订单已取消 | - |

#### 3.5.5 功能操作
- **新增订单**：选择产品、销售员，创建订单
- **编辑订单**：修改订单基本信息（待签约状态可编辑）
- **删除订单**：软删除（仅待签约状态可删除）
- **上传合同**：上传PDF/Word格式的电子合同
- **派单操作**：选择合适的白帽子分配订单
- **更换白帽子**：待履约状态可更换白帽子
- **取消订单**：填写取消原因
- **查看详情**：查看订单完整信息和流转记录
- **搜索筛选**：
  - 按订单ID搜索
  - 按产品名称搜索
  - 按订单状态筛选
  - 按销售人员筛选
  - 按时间范围筛选

#### 3.5.6 业务规则
- 订单金额默认取产品金额，可手动调整
- 上传合同后自动流转到"待派单"状态
- 分配白帽子后自动流转到"待履约"状态
- 取消订单需填写原因，不可恢复
- 已完成订单不可修改

## 十一、附录

### 11.1 核心业务规则总结

**用户角色**：
- 系统只有销售人员和合作伙伴两种角色
- 合作伙伴审核通过后可进行销售工作
- 一个销售人员只属于一个公司

**订单系统**：
- 订单中的"客户"不是系统用户，是销售人员手动填写的客户信息
- 订单完全由销售人员创建和管理
- 订单状态：签约中 → 派单中 → 履约中 → 已完成 / 已取消

**积分系统**：
- 销售人员完成订单获得积分
- 邀请他人注册获得积分奖励（邀请人100积分）
- 积分可兑换实体商品或虚拟商品

**优惠券系统**：
- 一个订单只能使用一张优惠券
- 优惠券需满足使用门槛
- 使用后立即标记为已使用

**地址管理**：
- 实体产品下单和兑换需要收货地址
- 必须有一个默认地址
- 最后一个地址不允许删除

### 11.2 关键交互说明

**头像选择器**：
- 点击"修改头像"展开/收起
- 8种颜色预设头像
- 4列网格布局
- 选中后自动收起并更新

**券码弹窗**：
- 虚拟商品兑换成功后自动弹出
- 显示券码和使用说明
- 支持复制券码

**邀请码二维码**：
- 从个人中心"我的邀请码"打开
- 显示邀请码文字和二维码
- 说明邀请奖励规则（100积分）


### 11.3 时间显示规则
- 相对时间：
  - 1分钟内：刚刚
  - 1小时内：X分钟前
  - 24小时内：X小时前
  - 7天内：X天前
  - 超过7天：显示完整日期
- 完整时间格式：`YYYY-MM-DD HH:mm`
