一、 视觉基础规范 (Design Tokens)
设计风格：微信小程序原生风格 (WeUI Like)。

色彩系统：

主色调 (Primary)：#0A7AFF —— 用于分段插件激活、主按钮、文字高亮。

价格/告警 (Danger)：#FF3B30 —— 用于优惠价格展示、过期/下架状态提示。

背景色 (Background)：#F2F2F7 —— 页面底色。

辅助色 (Success)：#34C759 —— 用于“已完成”、“审核通过”等状态。

文字颜色：#000000 (标题/正文)、#8E8E93 (次要描述/简介)。

字体规范：

标题：17px / Semibold

正文：14px / Regular

辅助：12px / Regular

二、 推荐前端框架与组件 (Cursor 配置)
前端框架：Vue3 + Uni-app (支持多端发布)

组件库：NutUI-Uniapp 或 Vant Weapp (成熟且符合微信视觉)

状态管理：Pinia (管理用户角色与订单状态)

三、 核心页面 UI 详细设计
1. 下单页面 (/pages/order/index)
设计目标：极简展示，去除所有非核心干扰项。

顶部区域：

Search 组件：搜索框背景色 #FFFFFF，圆角 8px。

筛选区域：

Tabs 组件：仅保留三个选项：[全部] [服务] [产品]。

注：禁止出现“已下架”标签。

列表区域：

Card 组件：

标题：左对齐，16px 粗体。

简介：下方紧跟 2-3 行灰色简述（去除了原有的类型标签）。

价格栏：左侧显示红色优惠价，右侧紧跟灰色划线原价。

特殊逻辑：

ID 为 12 的特定商品（如加密存储设备）：整体卡片不透明度 opacity: 0.6，并在右上角覆盖一个红色“已过期”的小角标，禁止点击进入详情。

2. 销售首页：我的订单 (/pages/sales/orders)
状态切换：[全部] [签约中] [履约中] [已完成] [已取消]。

订单卡片：展示订单 ID、客户名称、项目金额。

交互：点击卡片进入履约详情。

3. 角色选择与登录 (/pages/auth/login)
逻辑：微信一键登录后，若为新用户，强制进入“角色选择”页（合作伙伴 vs 销售人员）。

视觉：两个大的垂直排列卡片，点击即绑定角色。

五、 给 Cursor 的 Prompt 指令 (直接复制使用)
如果你正在使用 Cursor 的 Composer (Ctrl+I) 或 Chat (Ctrl+L)，请发送以下指令：

指令内容：
“请按照以下 UI 规范实现下单页面 (pages/order/index.vue)：

使用 Vue3 + NutUI-Uniapp 编写。

页面布局：

顶部 Search 搜索框。

接着是 Tabs 切换栏，选项固定为：['全部', '服务', '产品']，主色调 #0A7AFF。

列表内容：

数据源参考 PRD 中的产品列表（如企业安全评估、防火墙等）。

严禁显示任何分类标签（如‘安全评估’、‘网络安全’等副标题标签）。

每个产品卡片包含：产品名称（粗体）、简介（最多2行灰字）、价格。

价格显示：红色现价 + 灰色删除线原价。

特殊处理：

针对 ID 为 12 的‘加密存储设备’，设置不可点击状态，UI 整体透明度 0.6，并添加‘已过期’的红色 Tag。

色彩规范：

页面背景色 #F2F2F7，卡片背景色 #FFFFFF。”

六、 交互逻辑补充
下单流程：点击产品卡片进入配置页 -> 填写数量 -> 确认下单（弹出确认弹窗）。


刷新机制：列表支持下拉刷新，以获取最新的订单状态和积分余量。
