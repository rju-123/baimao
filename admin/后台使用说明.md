# 后台使用说明（产品与订单模块）

初始化脚本已执行后，按以下步骤使用后台。

## 1. 配置数据库

在 `admin` 目录下复制环境配置并修改为你的 MySQL 信息：

```bash
copy .env.sample .env
```

编辑 `.env`，确认 `[database]` 与当前 MySQL 一致，例如：

```ini
[database]
hostname = 127.0.0.1
database = fastadmin
username = root
password = 你的密码
hostport = 3306
prefix = fa_
```

## 2. 启动后台

在 `admin` 目录下启动（需已安装 PHP）。

**方式一：PHP 内置服务器（推荐本地开发）**

```bash
cd admin
php -S 127.0.0.1:8000 -t public
```

浏览器访问：`http://127.0.0.1:8000`。

**方式二：ThinkPHP 命令行（若支持）**

```bash
cd admin
php think run -p 8000
```

**方式三：Nginx / Apache**  
将网站根目录指向 `admin/public`。

## 3. 登录

- 首次使用需先执行 FastAdmin 的**安装流程**（访问安装页面完成数据库与管理员账号初始化）。
- 若已安装过，使用你自己的管理员账号登录。
- 演示站默认：用户名 `admin`，密码 `123456`（仅作参考，请以实际安装时设置的为准）。

## 4. 找到产品与订单菜单

- 登录后，在左侧菜单进入 **常规管理**。
- 其下会有：
  - **产品管理** → **产品列表**：产品增删改查。
  - **订单管理** → **订单列表**：订单增删改查。

若未看到上述菜单，请做第 5 步。

## 5. 分配菜单与权限

1. 进入 **权限管理** → **角色组**。
2. 编辑你当前管理员所在的角色（如“超级管理员”或“Admin group”）。
3. 在“菜单规则”中勾选：
   - **产品管理**（及其下的产品列表、添加、编辑、删除等若需要可一并勾选）。
   - **订单管理**（及其下的订单列表、添加、编辑、删除等若需要可一并勾选）。
4. 保存后刷新后台页面，左侧即可出现对应菜单。

## 6. 数据说明（后台 vs 前端 App）

- **后台（FastAdmin）**：使用 MySQL，表为 `fa_product`、`fa_order`，用于在后台维护产品与订单。
- **前端 uni-app**：当前对接的是 **NestJS API**（`api` 目录），该 API 使用 **SQLite**（`api/database.sqlite`），表为 `products`、`orders`。
- 因此：在后台添加/修改的产品与订单，**不会自动**出现在小程序/前端里；前端显示的是 API（SQLite）中的数据。
- 若希望前端展示后台维护的数据，需要后续将 NestJS API 改为连接同一 MySQL 库（使用 `fa_product`、`fa_order`），或做数据同步，可在此基础上再扩展。

## 7. 白帽子管理

在 MySQL 中执行白帽子表与菜单初始化（仅需执行一次）：

```bash
mysql -u root -p baimao_admin < admin/init_whitehat.sql
```

执行后刷新后台，左侧「常规管理」下会出现 **白帽子管理 → 白帽子列表**。可对白帽子（接单员）进行增删改查，编号格式为 H001、H002…，删除为软删除。

## 8. 小程序下单报 500 时（升级订单表）

若小程序在「确认下单」页点击「立即购买」后出现 **Internal server error (500)**，多半是订单表 `fa_order` 缺少 `product_customer` 列（表为旧版脚本创建）。

在 MySQL 中执行升级脚本（库名按实际修改）：

```bash
mysql -u root -p baimao_admin < admin/upgrade_fa_order_add_product_customer.sql
```

或在 Navicat 中打开 `admin/upgrade_fa_order_add_product_customer.sql` 执行。若提示「Duplicate column name」说明列已存在，可忽略。执行后重启 NestJS 接口服务，再在小程序中重试下单。

---

以上步骤完成后，即可在后台正常使用产品管理与订单管理功能。
