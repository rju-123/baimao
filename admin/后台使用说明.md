# 后台使用说明（产品与订单模块）

初始化脚本已执行后，按以下步骤使用后台。

## 1. 配置数据库

在 `admin` 目录下复制环境配置并修改为你的 MySQL 信息：

```bash
copy .env.sample .env
```

编辑 `.env`，确认 `[database]` 与当前 MySQL 一致。**必须使用 `baimao_admin`**，与 API 服务、初始化脚本一致，否则优惠券等数据无法互通：

```ini
[database]
hostname = 127.0.0.1
database = baimao_admin
username = root
password = 你的密码
hostport = 3306
prefix = fa_
```

## 2. 启动后台

在 `admin` 目录下启动（需已安装 PHP）。

**方式一：PHP 内置服务器（推荐本地开发）**

```bash
cd admin
php -S 127.0.0.1:8000 -t public
```

浏览器访问：`http://127.0.0.1:8000`。

**方式二：ThinkPHP 命令行（若支持）**

```bash
cd admin
php think run -p 8000
```

**方式三：Nginx / Apache**  
将网站根目录指向 `admin/public`。

## 3. 登录

- 首次使用需先执行 FastAdmin 的**安装流程**（访问安装页面完成数据库与管理员账号初始化）。
- 若已安装过，使用你自己的管理员账号登录。
- 演示站默认：用户名 `admin`，密码 `123456`（仅作参考，请以实际安装时设置的为准）。

## 4. 找到产品与订单菜单

- 登录后，在左侧菜单进入 **常规管理**。
- 其下会有：
  - **产品管理** → **产品列表**：产品增删改查。
  - **订单管理** → **订单列表**：订单增删改查。

若未看到上述菜单，请做第 5 步。

## 5. 分配菜单与权限

1. 进入 **权限管理** → **角色组**。
2. 编辑你当前管理员所在的角色（如“超级管理员”或“Admin group”）。
3. 在“菜单规则”中勾选：
   - **产品管理**（及其下的产品列表、添加、编辑、删除等若需要可一并勾选）。
   - **订单管理**（及其下的订单列表、添加、编辑、删除等若需要可一并勾选）。
4. 保存后刷新后台页面，左侧即可出现对应菜单。

## 6. 数据说明（后台 vs 前端 App）

- **后台（FastAdmin）**：使用 MySQL，表为 `fa_product`、`fa_order`、`fa_company`、`fa_sales` 等，用于在后台维护产品、订单、公司与销售人员。
- **NestJS API**（`api` 目录）：产品与订单读写 **MySQL**（`fa_product`、`fa_order`）；公司读写 **MySQL**（`fa_company`）；用户与地址等仍用 **SQLite**。小程序端销售登录/选公司时会同步到 MySQL 的 `fa_sales`，后台「销售人员管理」可查看。
- 因此：在后台维护的公司会直接体现在小程序「选择公司」列表；小程序添加的公司也会写入后台同一张表。销售在小程序选择公司后，会在后台「销售人员管理」中显示并可编辑所属公司。

## 7. 公司管理

在 MySQL 中执行公司表与菜单初始化（仅需执行一次）：

```bash
mysql -u root -p baimao_admin < admin/init_company.sql
```

执行后刷新后台，左侧「常规管理」下会出现 **公司管理 → 公司列表**。可对公司进行增删改查；后台维护的公司数据与小程序端「选择公司」列表一致（NestJS API 从同一 MySQL 库读取）。小程序「添加新公司」也会写入该表，在后台可见。

## 8. 销售人员管理

在 MySQL 中执行销售人员表与菜单初始化（仅需执行一次）：

```bash
mysql -u root -p baimao_admin < admin/init_sales.sql
```

执行后刷新后台，左侧「常规管理」下会出现 **销售人员管理 → 销售人员列表**。数据由小程序端销售登录或选择公司时自动同步，后台可查看手机号、姓名、所属公司及更新时间，并支持编辑「所属公司」。

## 9. 白帽子管理

在 MySQL 中执行白帽子表与菜单初始化（仅需执行一次）：

```bash
mysql -u root -p baimao_admin < admin/init_whitehat.sql
```

执行后刷新后台，左侧「常规管理」下会出现 **白帽子管理 → 白帽子列表**。可对白帽子（接单员）进行增删改查，编号格式为 H001、H002…，删除为软删除。

## 10. 优惠券管理

在 MySQL 中执行优惠券表与菜单初始化（仅需执行一次）：

```bash
mysql -u root -p baimao_admin < admin/init_coupon.sql
```

执行后刷新后台，左侧「常规管理」下会出现 **优惠券管理 → 优惠券模板**。可创建模板、编辑、发放。发放时选择公司，优惠券会写入 `fa_coupons`，小程序端该公司的销售/用户可在「个人中心 → 全部优惠券」中看到。

**若小程序看不到后台发放的优惠券**，请检查：

1. **数据库是否一致**：admin 的 `.env` 中 `database` 必须为 `baimao_admin`，与 NestJS API 使用同一库。若 admin 使用 `fastadmin`，优惠券会写入不同库，API 无法读取。
2. **公司是否匹配**：用户/销售需关联到发放优惠券时选择的公司（`fa_sales.companyId` 或用户表的 `companyId`）。
3. **调试接口**：访问 `GET /coupons/debug?userId=用户ID` 可查看 `companyIds`、`couponCount` 等，便于排查。

## 11. 小程序下单报 500 时（升级订单表）

若小程序在「确认下单」页点击「立即购买」后出现 **Internal server error (500)**，多半是订单表 `fa_order` 缺少 `product_customer` 列（表为旧版脚本创建）。

在 MySQL 中执行升级脚本（库名按实际修改）：

```bash
mysql -u root -p baimao_admin < admin/upgrade_fa_order_add_product_customer.sql
```

或在 Navicat 中打开 `admin/upgrade_fa_order_add_product_customer.sql` 执行。若提示「Duplicate column name」说明列已存在，可忽略。执行后重启 NestJS 接口服务，再在小程序中重试下单。

---

以上步骤完成后，即可在后台正常使用产品管理与订单管理功能。
